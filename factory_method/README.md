# Factory method

## Проблема
Необходимо реализовать логику работы одного класса, используя внутри другие, при этом код не должен зависеть
от того, с каким классом сейчас производится работа. Как пример, класс производящий определение имени и цвета
фигуры. И есть две фигуры, круг и прямоугольник.

## Решение
Создание объектов круга и прямоугольника не напрямую, а через фабричный метод. Получается у нас есть
общий класс оценщик фигур. У него есть фабричный метод который создает объект фигуры по ключу, передаваемому
как аргумент в данный метод, или мы делаем классы наследуемые от класса оценщика, и в данном
фабричном метода определяем какой конкретно класс фигуры должен быть реализован.

И чтобы не писать разную безнес логику под каждую отдельную фигуру, в каждой фигуре должен присутствовать
общий интерфейс получения имени или цвета, так вне зависимости какой объект фигуры был создан в фабричном
методе, бизнес логика опирается лишь на общие интерфейсы, которые есть у каждого класса фигуры.
